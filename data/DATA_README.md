> **This file documents the origin, collection method, and version of every
> file in the corpus. No data file may be used in experiments without an
> entry here.**

## Collection Methodology

The corpus is organized into three tiers:

- **Tier 1 (SOTA):** Established benchmark datasets from the academic
  literature, enabling direct comparison with published results. Primary
  source: Viotti (arXiv:2201.03051) SchemaStore-based evaluation.
- **Tier 2 (Canonical):** Real-world documents from authoritative sources
  (official specs, package registries, reference implementations). Selected
  for representativeness of how each format is used in production.
- **Tier 3 (Synthetic):** Machine-generated documents that stress-test
  specific constructs (deep nesting, dense comments, heavy anchors, etc.).
  Generated by deterministic scripts in `benchmarks/src/generate_synthetic.py`.

## Integrity Verification

All checksums stored in `data/checksums.sha256`. Before any experiment
execution, verify with:

```powershell
# PowerShell — verify all checksums
Get-Content data/checksums.sha256 | ForEach-Object {
    $parts = $_ -split '  '
    $expected = $parts[0]
    $file = $parts[1]
    $actual = (Get-FileHash -Algorithm SHA256 $file).Hash.ToLower()
    if ($actual -ne $expected) { Write-Error "MISMATCH: $file" }
    else { Write-Host "OK: $file" }
}
```
